1. Résumé Exécutif

La plateforme "Donation" est une application web innovante conçue pour faciliter et optimiser le processus de donation de médicaments, vêtements et autres biens essentiels. Ce projet vise à créer un pont entre les donateurs généreux et les personnes ou organisations nécessiteuses en proposant une solution numérelle complète, sécurisée et intuitive.

Cette plateforme permet aux utilisateurs de publier des annonces de donation, de consulter les donations disponibles, de gérer leur profil, et de tracer les transactions de manière transparente et traçable.

2. Introduction

Dans un monde en constante évolution, les besoins en aide humanitaire et en partage de ressources se font de plus en plus pressants. Malheureusement, les processus traditionnels de donation sont souvent compliqués, peu transparents et difficiles à gérer pour les deux parties impliquées.

C'est dans ce contexte que le projet "Donation" a été créé : offrir une plateforme numérique moderne et efficace qui simplifie les donations, améliore la traçabilité, et crée une communauté solidaire autour de l'entraide et du partage.

3. Description du Projet

Vue d'ensemble

La plateforme "Donation" est une application web full-stack qui permet :

  • Aux donateurs de créer des annonces de donation détaillées avec descriptions, photos et catégories
  • Aux receveurs de découvrir les donations disponibles dans leur région ou domaine d'intérêt
  • Aux administrateurs de modérer le contenu et de gérer les catégories de donation
  • À tous les utilisateurs de communiquer via un système de messagerie intégré
  • De maintenir un historique complet et traçable de toutes les transactions

Catégories de Donation

La plateforme supporte plusieurs catégories principales :

  • Médicaments et produits de santé : Médicaments, vitamines, suppléments, équipements médicaux
  • Vêtements : Habits, chaussures, accessoires de mode
  • Articles de maison : Meubles, électroménagers, décoration
  • Articles éducatifs : Livres, fournitures scolaires, équipements d'apprentissage
  • Autres : Catégorie flexible pour les donations spéciales

6. Fonctionnalités Principales

6.1 Authentification et Gestion des Utilisateurs

  • Inscription et connexion sécurisées avec validation d'email
  • Profils utilisateurs personnalisables (photo, bio, localisation)
  • Rôles et permissions (Donateur, Receveur, Administrateur)
  • Récupération de mot de passe sécurisée
  • Authentification deux facteurs optionnelle

6.2 Gestion des Donations

  • Création d'annonces de donation avec titre, description, photos, catégorie
  • Téléchargement d'images optimisées et sécurisées
  • Marquage de donations comme "Disponible", "En attente", "Donnée"
  • Édition et suppression d'annonces par le propriétaire
  • Historique complet des modifications

6.3 Découverte et Recherche

  • Filtrage par catégorie, localisation, date
  • Moteur de recherche textuel avancé
  • Liste des donations avec vignettes prévisualisées
  • Page de détail complète pour chaque donation
  • Notifications pour les nouvelles donations correspondant à la recherche

6.4 Communication

  • Système de messagerie interne entre utilisateurs
  • Notifications en temps réel pour les nouveaux messages
  • Historique des conversations persistant
  • Option de signalement/blocage des utilisateurs

6.5 Système de Catégories

  • Gestion complète des catégories par les administrateurs
  • Icônes personnalisées pour chaque catégorie
  • Catégories avec descriptions et guidelines
  • Possibilité de créer des sous-catégories

6.6 Tableau de Bord Utilisateur

  • Vue d'ensemble des donations actives de l'utilisateur
  • Historique des donations antérieures
  • Messages non lus et notifications
  • Statistiques et activité personnelle
  • Paramètres de compte et de confidentialité

6.7 Administration et Modération

  • Tableau de bord d'administration complet
  • Gestion des utilisateurs (activation, suspension, suppression)
  • Modération des donations (approbation, rejet, suppression)
  • Rapports et analytics sur l'utilisation
  • Gestion des catégories et de la configuration du site

7.2 Architecture du Projet

Structure MVC

Le projet suit le pattern MVC (Model-View-Controller) de Laravel :

  • Models : User, Donation, Category - représentent les données
  • Controllers : Logique métier et interactions avec les models
  • Views : Templates Blade pour le rendu HTML
  • Routes : Web (web.php) et API (api.php)

7.3 API RESTful

La plateforme expose une API RESTful pour :

  • Opérations CRUD sur les donations
  • Authentification et gestion des utilisateurs
  • Recherche et filtrage avancés
  • Système de messagerie
  • Notifications en temps réel (WebSockets optionnel)

8. Base de Données

8.1 Modèle de Données Conceptuel

Table : Users

Colonne Type Description
id BIGINT (PK) Identifiant unique de l'utilisateur
name VARCHAR(255) Nom complet de l'utilisateur
email VARCHAR(255) UNIQUE Adresse email unique
password VARCHAR(255) Mot de passe hashé (bcrypt)
phone VARCHAR(20) NULLABLE Numéro de téléphone optionnel
address VARCHAR(255) NULLABLE Adresse de localisation
bio TEXT NULLABLE Biographie courte de l'utilisateur
role ENUM('user', 'admin') Rôle de l'utilisateur
created_at, updated_at TIMESTAMP Métadonnées temporelles

Table : Categories

Colonne Type Description
id BIGINT (PK) Identifiant unique de la catégorie
name VARCHAR(255) Nom de la catégorie
description TEXT NULLABLE Description complète
icon VARCHAR(255) NULLABLE URL ou classe CSS de l'icône
created_at, updated_at TIMESTAMP Métadonnées temporelles

Table : Donations

Colonne Type Description
id BIGINT (PK) Identifiant unique de la donation
user_id BIGINT (FK) Référence à l'utilisateur donateur
category_id BIGINT (FK) Référence à la catégorie de donation
title VARCHAR(255) Titre de l'annonce
description TEXT Description détaillée de l'article
status ENUM('available', 'pending', 'given') État de la donation
image_path VARCHAR(255) NULLABLE Chemin vers l'image principale
location VARCHAR(255) NULLABLE Localisation du bien
created_at, updated_at TIMESTAMP Métadonnées temporelles

9. Sécurité et Conformité

9.1 Mesures de Sécurité Implémentées

  • Authentification : Passwords hashés avec bcrypt, sessions sécurisées
  • CSRF : Tokens CSRF sur tous les formulaires
  • Validation : Validation stricte de tous les inputs coté serveur
  • SQL Injection : Utilisation d'ORM (Eloquent) et requêtes préparées
  • XSS : Échappement automatique dans Blade templates
  • Autorisation : Middleware et policies pour contrôle d'accès
  • Rate Limiting : Protection contre les abus et attaques brute-force

9.2 Gestion des Données Sensibles

  • Stockage sécurisé des fichiers uploadés
  • Suppression complète des données lors de suppression de compte
  • Logs d'audit pour les actions critiques
  • Conformité RGPD pour la protection des données personnelles

9.3 Sécurité des Fichiers

  • Validation du type MIME des fichiers uploadés
  • Limitation de la taille des fichiers
  • Stockage en dehors de la racine web
  • Noms de fichiers randomisés pour éviter la prédiction
